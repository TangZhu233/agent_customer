name: project-update
description: 完成核心功能或Bug修复后，更新项目记忆文档。当你需要记录开发进度、更新项目摘要时调用。
---

按以下流程更新记忆文档：

1. **确认改动**: 先和用户确认刚才改了什么、解决什么问题、影响哪些文件
2. **生成记录**: 按 CLAUDE_PROGRESS.md 的模板格式生成一条新的进度记录，展示给用户确认：
   ```
   ### [YYYY-MM-DD HH:MM] 类型: 简短标题
   - **类型**: 功能开发 / Bug 修复 / 体验优化 / 基础设施 / 重构
   - **问题**: (为什么改)
   - **概述**: (做了什么)
   - **影响文件**: 列出所有改动文件
   - **验证**: 如何确认有效
   ```
3. **写入**: 用户确认后，追加到 CLAUDE_PROGRESS.md 的「进度时间线」末尾
4. **同步检查**: 判断 PROJECT_SUMMARY.md 是否需要更新：
   - 新增/删除文件 → 更新项目结构
   - 新增依赖 → 更新技术栈版本
   - 新增约束 → 更新关键约束
   - 新增功能 → 更新核心功能列表
5. **更新日期**: 修改 PROJECT_SUMMARY.md 顶部的最后更新时间

## 约束
- 写入前必须让用户确认记录内容
- 不记录琐碎改动（注释、格式化等）
